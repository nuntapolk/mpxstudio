[FEATURE] Tech Stack Catalog — ETA Section Expansion

  Backend — 6 ตาราง + 20 API Endpoints:
  - tech_catalog         : แกนหลัก Technology (name, vendor, category, tier, standard_status)
  - tech_versions        : Version tracking (major.minor.patch, release_type, EOL, lifecycle_phase)
  - tech_servers         : Server & Infrastructure inventory (hostname, OS, type, environment)
  - tech_usage           : Usage linking — App + Server (installed_version, upgrade_plan)
  - tech_vulnerabilities : CVE records (cve_id, cvss_score, severity, status, remediation)
  - tech_radar           : Technology Radar history per quarter (ring, quadrant, rationale)

  Business Rules:
  - Tech Tier:   Tier 1 (Strategic) → Tier 2 (Approved) → Tier 3 (Tolerated) → Tier 4 (Retire)
  - Tech Status: Approved | Restricted | Deprecated | Banned
  - Version Lifecycle: Active | Maintenance-only | EOL | Extended-support
  - Radar Rings: Adopt | Trial | Assess | Hold | Retire
  - Radar Quadrants: Languages & Frameworks | Platforms & OS | Tools & Build | Infrastructure & Cloud

  NVD CVE Auto-fetch:
  - ดึงข้อมูล CVE จาก NVD API 2.0 (nvd.nist.gov/rest/json/cves/2.0)
  - Background thread รัน auto-refresh ทุก 24 ชั่วโมง
  - Manual trigger: POST /api/tech/catalog/{id}/fetch-cve
  - Upsert by cve_id (ไม่ duplicate), support CVSS v3.1/v3.0/v2.0

  API Endpoints (20):
    GET    /api/tech/catalog                          List tech catalog (filter: category, tier, status, q)
    POST   /api/tech/catalog                          Create tech entry
    GET    /api/tech/catalog/{id}                     Get detail + versions + usage + CVEs + radar
    PUT    /api/tech/catalog/{id}                     Update tech
    DELETE /api/tech/catalog/{id}                     Delete tech (cascade)
    GET    /api/tech/catalog/{id}/versions            List versions
    POST   /api/tech/catalog/{id}/versions            Add version
    PUT    /api/tech/versions/{id}                    Update version
    DELETE /api/tech/versions/{id}                    Delete version
    GET    /api/tech/servers                          List servers (filter: env, status, q)
    POST   /api/tech/servers                          Create server
    PUT    /api/tech/servers/{id}                     Update server
    GET    /api/tech/usage                            List usage (filter: tech_id, app_id, server_id)
    POST   /api/tech/usage                            Add usage record (App or Server)
    PUT    /api/tech/usage/{id}                       Update usage
    DELETE /api/tech/usage/{id}                       Delete usage
    GET    /api/tech/vulnerabilities                  List CVEs (filter: tech_id, severity, status)
    POST   /api/tech/catalog/{id}/fetch-cve           Trigger NVD fetch (background)
    PUT    /api/tech/vulnerabilities/{id}             Update CVE status/remediation
    GET    /api/tech/radar                            Radar entries (filter: quarter, ring, quadrant)
    POST   /api/tech/radar                            Add radar entry
    GET    /api/tech/dashboard                        KPIs + by_category + by_tier + CVE breakdown

  Frontend — 4 Views ใน ETA section:
  - 📦 Tech Catalog: Table + right slide-over panel พร้อม 4 tabs (Versions, Usage, CVEs, Radar)
    Create/Edit modal, Add Version modal, Add Usage modal (App/Server), Fetch NVD button
  - 🎯 Tech Radar: Full SVG Radar chart 800×800px, 4 rings × 4 quadrants, blips interactive
    Hover tooltip แสดง ring/quadrant/version/usage count, Quarter filter, Blip list table
  - 🛡️ Vulnerabilities: KPI cards (Critical/High/Medium/Low/Total), CVE table + status filter,
    Update CVE status modal, "Fetch All CVEs" bulk button
  - 🖥️ Server Inventory: Server table + filter by env/status, Add/Edit server modal

  Sidebar (techarch group) — ลำดับใหม่:
  - Tech Overview, Tech Catalog, Tech Radar, Vulnerabilities, Server Inventory,
    Tech Standard Catalog, Tech Product Catalog, App Tech Map, Tech Gap

[FEATURE] ARB Lite — Architecture Review Board Module (New Sidebar Section)

  Backend — 9 ตาราง + 24 API Endpoints:
  - arb_requests          : ARB Review Request หลัก (code, type, status, review_level)
  - arb_request_applications: linked applications ต่อ request
  - arb_impact_profile    : Impact flags 13 ด้าน (Business/Data/App/Tech/Security/
                            Integration/Compliance + has_pii, internet_facing,
                            new_integration, new_vendor, new_technology, expected_exception)
  - arb_reviewers         : Reviewer assignment per request
  - arb_comments          : Review comments (domain, severity, type)
  - arb_findings          : Formal findings (severity: Low/Medium/High/Critical)
  - arb_decisions         : ARB Decision record (Approve/Conditionally/Rework/Reject)
  - arb_actions           : Follow-up action items + tracking
  - arb_recommendations   : Auto-generated artifact/ESA ABB-SBB recommendations

  Business Rules:
  - Auto Review Level Classification:
      * Auto-pass  → ทุก impact = None + ไม่มี flag ใดเป็น True
      * Desk Review → impact ≤ Medium + flag ≤ 2 ตัว
      * Formal Review → มี High/Critical impact หรือ flag ≥ 3 ตัว
  - Auto Recommendation Engine:
      * data_impact → แนะนำ Data Flow Diagram + Data Dictionary
      * security_impact / has_pii → แนะนำ Security Assessment + ESA ABB/SBB
      * integration_impact → แนะนำ Interface Specification
      * new_technology → แนะนำ Technology Assessment
      * internet_facing → แนะนำ Internet Risk Assessment
      * expected_exception → แนะนำ Exception Approval Form

  API Endpoints (24):
    POST   /api/arb/requests                          Create new ARB request
    GET    /api/arb/requests                          List requests (filter/search)
    GET    /api/arb/requests/{id}                     Get request detail
    PUT    /api/arb/requests/{id}                     Update request
    POST   /api/arb/requests/{id}/submit              Submit for review
    POST   /api/arb/requests/{id}/status              Change status
    DELETE /api/arb/requests/{id}                     Delete request
    POST   /api/arb/requests/{id}/reviewers           Assign reviewer
    DELETE /api/arb/requests/{id}/reviewers/{rid}     Remove reviewer
    POST   /api/arb/requests/{id}/comments            Add comment
    POST   /api/arb/requests/{id}/findings            Add finding
    PUT    /api/arb/findings/{id}                     Update finding
    POST   /api/arb/requests/{id}/decision            Issue decision
    POST   /api/arb/requests/{id}/actions             Add action
    PUT    /api/arb/actions/{id}                      Update action status
    GET    /api/arb/requests/{id}/recommendations     Get recommendations
    POST   /api/arb/requests/{id}/generate-recommendations  Regenerate
    GET    /api/arb/dashboard                         Dashboard KPIs + stats
    GET    /api/arb/requests/export/csv               Export CSV

  Frontend — 2 Views + full workflow:
  - ARB section ใน Sidebar (ต่อจาก Core Products, ก่อน Meta Data)
  - 🏛️ ARB Dashboard: KPI cards (total/open/decision), by-status/level/decision
    charts, overdue action count, recent requests table
  - 📋 ARB Requests: Request list พร้อม search + filter (status/level),
    Detail slide-over panel แสดง: header, impact profile, reviewers,
    recommendations, findings, actions, comments, decision
  - Create Request Modal: title, type, owner, target date, objective, summary,
    impact matrix (6 dropdowns + 6 checkbox flags)
  - Add Finding Modal: domain, severity, description, recommended action, owner, due date
  - Add Action Modal: description, type, owner, due date, required evidence
  - Issue Decision Modal: decision type, summary, rationale, key risks, next steps
  - Inline actions: Submit / Change Status / Add Comment / Close Action / Regen Recs

  Permissions:
  - menu key "arb" เพิ่มในทุก editor account ทั้ง 13 accounts (ไม่รวม viewer, vendor)
  - menus: ['arb-dashboard','arb-requests']
